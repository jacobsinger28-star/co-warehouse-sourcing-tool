## What this session did

**Built `tools/scaffold_index.py`** — bridges the `deal-index.csv` and the `/deals/<deal_id>/`
folder archive. The validator (`validate_index.py`) checks the CSV in isolation; this checks it
against reality on disk. Two faces of one folder walk:

- **scaffold (default):** walks the archive and prints a skeleton `deal-index.csv` with `deal_id`
  (from each folder name) and `source_docs` (from the files in it) filled, every other column
  blank. Mechanizes the two most tedious intake columns for hundreds of folders — and invents
  nothing (blanks stay blank, per the no-fabrication rule). Pipe to a file, fill the blanks, gate
  with the validator. The two tools bookend intake.
- **`--check INDEX`:** reconciles an existing index against the folders. **Errors:** a deal folder
  with no index row (undocumented deal); `source_docs` citing a file not in the folder (dangling
  citation). **Warnings:** an index row whose `deal_id` has no folder (fine if Pipedrive-only); a
  file in a folder not listed in `source_docs` (a new LOI nobody indexed); non-slug folder name;
  empty folder. Run it right after the validator before every re-upload.

Read-only against the archive — never writes to it, safe on the live shared drive. Stdlib only.
Reuses `CANONICAL_HEADER` + `DEAL_ID_RE` from `validate_index.py` so the schema stays single-source.
Exit 0 clean / 1 errors (or warnings under `--strict`) / 2 archive unreadable.

- `tools/test_scaffold.py` — **23 tests, all green.** Drives the pure core (reconcile /
  skeleton_rows / lint_folders) with in-memory dicts, plus a tempdir archive for the folder walk +
  CLI end-to-end.
- Wired into the workflow: `tools/README.md` (full usage), `deal-archive-spec.md` (scaffold
  shortcut in intake order + reconcile in the quality-bar gate), `how-to-use.md` (both checks before
  every upload/refresh), root `README.md` (tree + next-moves).

**Compiled `docs/STAGE-0-INPUTS-NEEDED.md`** — the runbook of every human input (access, decision,
information, sample data, confirmation) needed to unblock Stage 0, grouped by owner with a top-3
